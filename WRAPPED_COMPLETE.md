## 🏗️ Архитектура

### Сбор данных
- Каждое прослушивание автоматически записывается в `play_history`
- Хранится: user_id, track_id, timestamp, duration, platform, completed
- Индексы обеспечивают быструю выборку

### Генерация статистики
- SQL агрегация через GROUP BY, COUNT, SUM
- Расчёт топов через ORDER BY play_count DESC LIMIT 10
- Бейджи вычисляются на основе порогов (5000+ минут = Меломан)
- Percentile через сравнение с другими пользователями

### Frontend Flow
1. Компонент загружает доступные годы
2. Пользователь выбирает год
3. Запрос к API генерирует статистику
4. Отображение через слайды с навигацией
5. Кнопка "Поделиться" создаёт публичную ссылку
